PROJECTS

Bin Packer

 In the online MMO Elite Dangerous, there is a colonization system that allows the construction of stations in unclaimed systems. The construction of stations requires tens of thousands, or up to hundreds of thousands of materials.

 Finding the markets, planning each trip, and keeping track of the quantity required for each material felt tedious. In response, I created the first version of Bin Packer. The first version only took in a text file that matched a set format, and planned the materials to buy for each trip. However, that only solved the planning and the material quantity tracking; I still had to find each market.

 Each update aimed at automating a step that was previously done manually. I started by adding the ability to search the game's Journal files for the list of required materials. Then, I made it search online for markets that have the required materials. Finally, I made an algorithm to plan a route from the station, to the markets, and back to the station.


Orbits

 While listening to music, I came across a music video titled " co·a·les·cence ". I not only liked the music, but loved the visuals. Seeing planets with different landmasses, some with or without asteroid rings, some with no moons, while others had many moons. Watching all the moons orbiting the main planet. I wanted to make something similar, so it can run on a monitor or act as a background to stare at whenever I'm bored or lost.

 When I began planning how to create it, I knew that I'd have to get the orbit simulation working before I could focus on the finer details. However, I couldn't watch numbers changing and tell that a simulation was working, so I had to add something to watch. Since I was aiming to create planets, I tried to make the bodies circular, so I settled on using a simplistic algorithm to get circles that worked. I then began focusing on the logic for the simulation.

 I created a list of all the bodies present in the system, including their masses, radii, velocities, and colors. Then, I passed the list over to an algorithm that used each body's mass to calculate the force imparted onto the other bodies while ignoring itself. Once all the forces had been calculated and combined, I converted them into acceleration and updated the body's velocity. With that, all the bodies affected each other, leading to orbits (or the flinging of bodies).

 However, I later came across a bug caused by code that kept the simulation's motion stable between frames. The code was used to determine the time between frames, then multiply the forces by it to get an accurate and stable motion. The bug occurs when the computer lags or when the simulation is paused and then resumed. That makes the time difference large enough to send bodies into higher orbits, or more likely, launch them off the screen, never to be seen again.

 As this project is currently abandoned, I am not going to fix this. But when I do return, I will fix the bug and attempt to add even more features, such as centering the view on a body.
